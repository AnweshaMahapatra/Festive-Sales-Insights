🔴 The Meeting Nobody Wanted to Have

Every festive season, AtliQ Mart runs promotions. Discounts, cashbacks, bundle deals — the works. The stores fill up. The tills ring. And then the season ends, and someone has to write the report.

That report usually says: revenue went up.

And technically, it did.
Pre-promotion revenue was ₹141 million.
Post-promotion was ₹296 million.
Incremental revenue: ₹155 million. A 211% lift.

Numbers that look spectacular on a slide.

But somewhere in the planning room for the next campaign, a quieter question starts circulating:

Did the promotions actually cause that? And if they did — which ones?

That question is harder to answer than it looks. And answering it honestly required building something that didn't exist yet.

🟡 Two Products That Changed the Conversation

Before any dashboard existed, the first step was SQL — raw queries against transaction data.

One of the first questions:

List products priced over ₹500 that were featured in a BOGOF promotion.

🧾 Result:
Atliq Waterproof Immersion Rod — ₹1,190
Atliq Double Bedsheet Set — ₹1,020

Buy One Get One Free on ₹1,000+ products.

That is effectively a ~50% discount impact per unit.

The key question:

Does volume uplift justify margin loss?

📊 Final insight:
Atliq Waterproof Immersion Rod → 266.19% IR% (Highest across campaigns)
Strategy worked — but only for specific categories.

🟠 What the Dashboard Was Actually For

This was never about charts.

It was about answering:

Which promotions actually worked?
Which cities lost money while looking successful?
Which categories respond vs ignore discounts?
Where is revenue real vs artificially inflated?
🎯 What changed:
Store managers can now compare IR% by location
Category managers can see elasticity differences
Campaign planners can redesign strategy before execution
🧠 Final Insight

The dashboard didn’t create answers.

It simply made one thing unavoidable:

Some promotions were driving growth.
Some were destroying it.
And before this, nobody could clearly see which was which.
